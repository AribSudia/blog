## المشكلة

بعد التحديث الأخير لـ OpenClaw، صار النظام يطلب موافقتك على **كل أمر** قبل تنفيذه:

```
Exec approval needed
```

هذا السلوك الجديد مُصمم للأمان، لكنه قد يكون مزعج إذا كنت تثق بالنظام وتريده يشتغل بحرية.

---

## الحل السريع

شغّل هذا الأمر في Terminal:

```bash
openclaw config set tools.exec.security full && openclaw gateway restart
```

**وخلاص!** 🎉

النظام يرجع يشتغل تلقائياً بدون ما يسألك كل مرة.

---

## شرح الأمر

| الجزء | الوظيفة |
|-------|---------|
| `openclaw config set` | تعديل إعدادات OpenClaw |
| `tools.exec.security` | إعداد صلاحيات تنفيذ الأوامر |
| `full` | صلاحيات كاملة (بدون موافقة) |
| `openclaw gateway restart` | إعادة تشغيل النظام لتطبيق التغييرات |

---

## مستويات الأمان المتاحة

| المستوى | الوصف |
|---------|-------|
| `deny` | ممنوع تنفيذ أي أمر |
| `allowlist` | فقط الأوامر المحددة مسبقاً |
| `full` | صلاحيات كاملة (الافتراضي القديم) |

---

## ليش التغيير؟

فريق OpenClaw أضاف طبقة أمان إضافية بحيث:
- الأوامر الحساسة تحتاج موافقة
- تقدر تراجع الأمر قبل تنفيذه
- حماية من الأخطاء غير المقصودة

لكن إذا كنت مستخدم متقدم وتثق بالنظام، الخيار `full` يرجّع السلوك القديم.

---

## ملاحظات مهمة

⚠️ **تحذير:** استخدم `security: full` فقط إذا:
- أنت الوحيد اللي يستخدم النظام
- تفهم المخاطر المحتملة
- عندك نسخ احتياطية للبيانات المهمة
